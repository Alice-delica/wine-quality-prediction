# wine-quality-prediction
Dataset yang digunakan adalah dataset Wine Quality yang berisi fitur-fitur kimiawi dari anggur merah dan putih serta nilai kualitasnya. Data_testing.csv yang berisi fitur kimiawi anggur, namun tanpa label kualitas, lalu akan diprediksi nilai kualitas untuk data testing tersebut. Dengsn membangun model klasifikasi machine learning yang mampu memprediksi kualitas anggur berdasarkan karakteristik fisikokimia. Analisis ini menggunakan dataset yang mencakup berbagai fitur kimiawi seperti tingkat keasaman, kandungan gula, kadar alkohol, dan pH.

## Alur Kerja
1. **Persiapan Data**
Langkah awal untuk merapikan data sebelum mulai dihitung oleh komputer, yang dimulai dengan memasukkan file data training dan testing ke dalam program Python. Dalam tahap ini, saya memisahkan antara informasi kimiawi anggur yang akan dipelajari dengan target skor kualitas yang ingin ditebak. Hal yang paling penting adalah saya menyimpan kolom Id dari data testing secara terpisah agar tidak ikut diolah, sehingga nantinya hasil prediksi saya tidak tertukar dan tetap berpasangan dengan identitas sampel anggur yang benar saat dikumpulkan. Dengan persiapan yang matang, saya memastikan bahwa semua kolom yang digunakan untuk melatih model sudah sama persis dengan kolom yang ada pada data ujian.
2. **Pembersihan Data**
Tahap pembersihan data dilakukan untuk memastikan kualitas data sebelum masuk ke tahap pemodelan agar hasil prediksinya lebih akurat. Pertama, saya menangani data yang kosong dengan cara mengisi kekosongan tersebut menggunakan nilai median, karena median tidak mudah terpengaruh oleh angka-angka ekstrem yang mungkin ada dalam data kimiawi anggur. Selanjutnya, saya melakukan Feature Scaling atau penyamaan skala untuk semua informasi kimiawi yang ada. Hal ini penting karena setiap variabel memiliki rentang nilai yang berbeda jauh, misalnya ada yang bernilai ratusan dan ada yang hanya nol koma sehingga dengan menyamakan skalanya, model tidak akan bingung dan bisa memberikan bobot yang adil pada setiap informasi saat menentukan kualitas anggur nanti.
3. **Pembuatan Model**
Pada tahap Pembuatan Model, saya menggunakan algoritma Random Forest Classifier untuk melatih data agar komputer bisa belajar mengenali pola kualitas anggur. Sebelum memprediksi data ujian yang sesungguhnya, saya membagi data pelatihan menjadi dua bagian, yaitu data untuk belajar dan data untuk tes internal (validasi), guna memastikan model tidak hanya menghafal data tetapi benar-benar bisa memberikan jawaban yang tepat pada data baru. Saya memilih Random Forest karena model ini bekerja seperti pohon keputusan yang menggunakan banyak data sebelumnya untuk digabungkan pendapatnya, sehingga hasil prediksinya lebih stabil, akurat, dan tidak mudah terkecoh oleh angka-angka aneh yang mungkin muncul dalam data kimiawi anggur.
4. **Prediksi Data Uji**
Tahap Prediksi Data Uji adalah saat di mana model yang sudah dilatih diterapkan pada data asli yang belum memiliki label kualitas. Setelah model mempelajari pola dari data pelatihan, saya memasukkan fitur kimiawi dari data_testing.csv untuk menebak skor kualitasnya. Hasil dari tebakan model ini kemudian disatukan kembali dengan kolom Id yang telah disimpan sebelumnya agar setiap baris data memiliki identitas yang jelas. Hasil akhirnya disimpan dalam file CSV yang hanya berisi dua kolom, yaitu identitas sampel dan hasil prediksi kualitasnya
## Hasil Evaluasi
- **Akurasi:** 0.5813953488372093
Nilai akurasi 0,58 berarti model berhasil menebak kualitas anggur dengan benar tepat setengah dari total data pengujian. Meskipun angka ini terlihat rendah, dalam konteks klasifikasi multi-kelas (di mana ada 5-6 kategori kualitas), akurasi 50% sebenarnya sudah cukup baik. Nilai akurasi model yang tidak terlalu tinggi juga diakibatkan oleh imbalanced antar kelas.
- **Confusion Matrix**
Menunjukkan bahwa model cenderung memiliki performa yang kuat pada kelas mayoritas, yakni kualitas 6 dan 7, namun gagal total dalam mengenali kelas ekstrem seperti kualitas 4 atau 8 karena tidak adanya prediksi yang berhasil mendarat tepat di diagonal untuk kelas tersebut. Banyaknya kesalahan prediksi pada data dekat (misalnya kualitas 6 yang terdeteksi sebagai 7 atau sebaliknya) mengindikasikan adanya kemiripan karakteristik kimiawi yang sangat tipis antar tingkatan tersebut, serta adanya pengaruh ketidakseimbangan jumlah data (class imbalance) yang membuat model lebih condong menebak nilai yang paling sering muncul dalam dataset pelatihan.
